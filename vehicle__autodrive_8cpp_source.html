38/*
39 * This file contains code that allows a vehicle to be driven by an in-game character (most
40 * likely the player character) to some remote destination, following a path of adjacent overmap
41 * tile (OMT) coordinates, while avoiding any obstacles along the way.
42 *
43 * The main entry point is do_autodrive(), which is intended to be called from the do_turn() of
44 * a long-running activity. This will cause the driver character to perform one or more driving
45 * actions (as long as they have enough moves), which could be steering and/or changing the
46 * cruise control speed. The driver may also do nothing if the vehicle is going in the right
47 * direction.
48 *
49 * Most of the logic is inside the private implementation class autodrive_controller, which is
50 * instantiated at the beginning of an autodrive activity and cleaned up at the end. This class
51 * caches information about the vehicle, driver and the local terrain in order to minimize the
52 * amount of computation required on each invocation of do_autodrive().
53 *
54 * Upon reaching a new overmap tile, as well as once initially, the autodrive algorithm will
55 * try to find a path that takes the vehicle from its current position through the current
56 * and next OMT in order to reach the 2nd next OMT in the overall path to destination (which
57 * is obtained from driver.omt_path). Thus we always plan ahead 2 OMTs worth of navigation,
58 * but we only follow the first half of the route, then recompute a new route for the next
59 * 2 OMTs. This ensures that we don't drive up to the edge of an OMT only to get stuck due
60 * to a large obstacle that can't be driven around from that position (a dead end) and instead
61 * we always choose to cross OMT boundaries in a position from which we are guaranteed to be
62 * able to continue onward.
63 *
64 * The pathfinding algorithm being employed is A*. The graph being searched is made up of nodes
65 * denoted by x and y coordinates as well as a direction of travel. Since steering changes the
66 * vehicle's direction in increments of 15 degrees there are 360/15 possible orientations. By
67 * limiting navigation to 2 OMTs at a time we have a maximum search space of 2*24*24*24 nodes.
68 * In practice, A* finds a good path after visiting fewer than 50 nodes due to the heuristic
69 * function that estimates which nodes are most likely to lead to the destination (which is
70 * based on proximity to the "exit zone" leading to the next OMT and whether the vehicle is
71 * pointing towards the middle of that zone). Although there can be many paths that we could
72 * take to get through to the next OMT, we just take the first one we find since it tends to
73 * be the most direct and is usually near-optimal. When the way is blocked we'll either give
74 * up after exhausting the search space or after we hit a limit on the search length.
75 *
76 * In order to build the search graph for A*, we consider every possible map tile and orientation
77 * in the search zone and we figure out whether the vehicle's pivot point could be placed there
78 * without the vehicle overlapping some obstacle (a collision-free placement). The pivot point
79 * is the part of the vehicle that moves the least while steering and is usually between the
80 * wheels of the vehicle, closer to the non-steerable ones. The pathfinding problem reduces to
81 * finding a path for the pivot point to follow through the set of possible positions and
82 * orientations (nodes). In order to figure out the possible connections between nodes (the
83 * edges of the graph) we consider a limited set of actions that the driver can perform at
84 * each location: nothing, steer left or steer right (possibly more than once). Given a current
85 * node, current speed and amount of steering we can predict where the vehicle's pivot point
86 * will end up in 1 turn and connect the 2 nodes.
87 *
88 * In order to keep the search space small we only consider paths traveled at constant speed
89 * (otherwise we'd have to add speed as an extra attribute in every node). When we need to
90 * compute a new path we first try to use the maximum safe autodrive speed (which is typically
91 * much less than the vehicle's max speed). If we can't find a path we cut the speed in half and
92 * try again, repeating until we succeed or reach the minimum speed. When the target speed is
93 * different from the current speed of the vehicle we try to estimate how long it will take to
94 * reach that speed and take that into account when simulating the first few edges (they'll be
95 * shorter or longer than the rest).
96 *
97 * Once we have generated a sequence of nodes to follow, we do not invoke A* again until we reach
98 * the next OMT. We simply cache the path and on every subsequent call to do_autodrive() we
99 * check where we are on that path, what node we're supposed to reach next and what action we
100 * should use to get there. However, sometimes the simplified simulation used to compute the
101 * search graph fails to accurately predict where the vehicle will end up. This could be because
102 * there was too much error in an approximation (such as time to accelerate) or the driving
103 * conditions changed (ran over a mound of dirt that interfered with steering). When such a
104 * discrepancy is detected, autodrive will fall back to "safe mode", cutting the speed to the
105 * minimum (1 tile per second) and recomputing the path.
106 *
107 * Since the navigation graph is cached and not updated in response to dynamic obstacles (such
108 * as animals) autodrive will also perform collision detection at every turn, before taking
109 * any action that may end the turn. If a possible collision is detected autodrive will enter
110 * "safe mode", reducing speed and recomputing. If the speed is already at minimum it will
111 * abort instead with a warning message.
112 *
113 * In most cases when autodrive aborts it will not automatically stop the vehicle. Stopping
114 * may be undesirable since nearby enemies could get into the vehicle. It is up to the player
115 * to either stop or continue driving manually for a while. However, when aborting due to a
116 * collision (which happened despite collision detection, i.e. there was a simulation discrepancy
117 * near an obstacle) the brakes will be engaged.
118 *
119 * In order to prevent exploiting the system to navigate in the dark, autodrive will also
120 * perform visibility checks. Not being able to see anything in front of the vehicle will
121 * immediately cancel (or fail to start) autodrive. If the driver has only partial visibility
122 * of the front of the vehicle, safe mode will engage limiting the speed. Furthermore, the
123 * driver needs to either see or have map memory for the map tiles in the current OMT; all
124 * unseen tiles will be treated as obstacles during pathfinding. In practice, this means
125 * that driving at night should only be possible through familiar terrain with limited
126 * lights / night vision or through unfamiliar terrain but with good headlights / excellent
127 * night vision.
128 */

479/*
480 * Computes atan2(dy, dx) in "orientation" units. Optimized to use only integer
481 * arithmetic and table lookups since it's used in the inner loop of A*.
482 */
483// NOLINTNEXTLINE(cata-xy)
484static orientation approx_orientation( int dx, int dy )
485{
486 orientation ret = orientation::d0;
487 if( dy < 0 ) {
488 dx *= -1;
489 dy *= -1;
490 ret += quad_rotation::d180;
491 }
492 if( dx < 0 ) {
493 std::swap( dx, dy );
494 dy *= -1;
495 ret += quad_rotation::d90;
496 }
497 if( dy > dx ) {
498 ret = ret + quad_rotation::d90 - approx_orientation( dy, dx );
499 } else if( dy > 0 ) {
500 static const auto atan_table = [] {
501 constexpr int table_size = 101;
502 std::array<orientation, table_size> table;
503 for( int i = 0; i < table_size; i++ )
504 {
505 table[i] = to_orientation( units::from_radians( std::atan( 1.0 * i /
506 ( table_size - 1 ) ) ) );
507 }
508 return table;
509 }();
510 ret = ret + atan_table[ dy * 100 / dx ];
511 }
512 return ret;
513}

577vehicle_profile vehicle::autodrive_controller::compute_profile( orientation facing ) const
578{
579 vehicle_profile ret;
580 tileray tdir( to_angle( facing ) );
581 ret.tdir = tdir;
582 std::map<int, std::pair<int, int>> extent_map;
583 const point pivot = driven_veh.pivot_point();
584 for( const vehicle_part &part : driven_veh.parts ) {
585 if( part.removed ) {
586 continue;
587 }
588 tripoint pos;
589 driven_veh.coord_translate( tdir, pivot, part.mount, pos );
590 if( extent_map.find( pos.y ) == extent_map.end() ) {
591 extent_map[pos.y] = { pos.x, pos.x };
592 } else {
593 auto &extent = extent_map[pos.y];
594 extent.first = std::min( extent.first, pos.x );
595 extent.second = std::max( extent.second, pos.x );
596 }
597 }
598 for( const auto &extent : extent_map ) {
599 const int y = extent.first;
600 const int x_min = extent.second.first;
601 const int x_max = extent.second.second;
602 for( int x = x_min; x <= x_max; x++ ) {
603 ret.occupied_zone.emplace_back( x, y );
604 }
605 }
606 for( int part_num : driven_veh.rotors ) {
607 const vehicle_part &part = driven_veh.cpart( part_num );
608 const int diameter = part.info().rotor_diameter();
609 const int radius = ( diameter + 1 ) / 2;
610 if( radius > 0 ) {
611 tripoint pos;
612 driven_veh.coord_translate( tdir, pivot, part.mount, pos );
613 for( tripoint pt : points_in_radius( pos, radius ) ) {
614 ret.occupied_zone.emplace_back( pt.xy() );
615 }
616 }
617 }
618 // figure out the maximum amount of displacement that can happen from advancing the
619 // tileray a single step by advancing it a bunch of times and reducing the x and y
620 // components of the displacement to at most 1
621 constexpr int far_advance = 100;
622 tdir.advance( far_advance );
623 const point increment( signum( tdir.dx() ), signum( tdir.dy() ) );
624 // compute the subset of vehicle points that would move to previously unoccupied spots
625 // (a.k.a. "moving first") when the vehicle moves one step; these are the only points
626 // we need to check for collision when the vehicle is moving in this direction
627 const std::unordered_set<point> occupied_set( ret.occupied_zone.begin(), ret.occupied_zone.end() );
628 for( const point &pt : ret.occupied_zone ) {
629 if( occupied_set.find( pt + increment ) == occupied_set.end() ) {
630 ret.collision_points.emplace_back( pt );
631 }
632 }
633 return ret;
634}

640bool vehicle::autodrive_controller::check_drivable( tripoint pt ) const

642 const map &here = get_map();
643
644 // check if another vehicle is there; tiles occupied by the current
645 // vehicle are evidently drivable
646 const optional_vpart_position ovp = here.veh_at( pt );
647 if( ovp ) {
648 // Known corner case: some furniture can be driven over, but will collide with
649 // wheel parts; if the vehicle starts over such a furniture we'll mark that tile
650 // as safe and may collide with it by turning; however if we mark it unsafe
651 // we'll have no viable paths away from the starting point.
652 return &ovp->vehicle() == &driven_veh;
653 }
654
655 const tripoint_abs_ms pt_abs( here.getabs( pt ) );
656 const tripoint_abs_omt pt_omt = project_to<coords::omt>( pt_abs );
657 // only check visibility for the current OMT, we'll check other OMTs when
658 // we reach them
659 if( pt_omt == data.current_omt ) {
660 // driver must see the tile or have seen it before in order to plan a route over it
661 if( !driver.sees( pt ) ) {
662 if( !driver.is_avatar() ) {
663 return false;
664 } else if( driver.as_avatar()->get_memorized_tile( pt_abs.raw() ) == mm_submap::default_tile ) {
665 // apparently open air doesn't get memorized, so pretend it is or else
666 // we can't fly helicopters due to the many unseen tiles behind the driver
667 if( !( data.air_ok && here.ter( pt ) == t_open_air ) ) {
668 return false;
669 }
670 }
671 }
672 }
673
674 // check for creatures
675 // TODO: padding around monsters
676 Creature *critter = g->critter_at( pt, true );
677 if( critter && driver.sees( *critter ) ) {
678 return false;
679 }
680
681 // don't drive over visible traps
682 if( here.can_see_trap_at( pt, driver ) ) {
683 return false;
684 }
685
686 // check for furniture that hinders movement; furniture with 0 move cost
687 // can be driven on
688 const furn_id furniture = here.furn( pt );
689 if( furniture != f_null && furniture.obj().movecost != 0 ) {
690 return false;
691 }
692
693 const ter_id terrain = here.ter( pt );
694 if( terrain == t_null ) {
695 return false;
696 }
697 // open air is an obstacle to non-flying vehicles; it is drivable
698 // for flying vehicles
699 if( terrain == t_open_air ) {
700 return data.air_ok;
701 }
702 const ter_t &terrain_type = terrain.obj();
703 // watercraft can drive on water
704 if( data.water_ok && terrain_type.has_flag( TFLAG_SWIMMABLE ) ) {
705 return true;
706 }
707 // remaining checks are for land-based navigation
708 if( !data.land_ok ) {
709 return false;
710 }
711 // NOLINTNEXTLINE(bugprone-branch-clone)
712 if( terrain_type.movecost <= 0 ) {
713 // walls and other impassable terrain
714 return false;
715 } else if( terrain_type.movecost == 2 || terrain_type.has_flag( "NOCOLLIDE" ) ) {
716 // terrain with neutral move cost or tagged with NOCOLLIDE will never cause
717 // collisions
718 return true;
719 } else if( terrain_type.bash.str_max >= 0 && !terrain_type.bash.bash_below ) {
720 // bashable terrain (but not bashable floors) will cause collisions
721 return false;
722 } else if( terrain_type.has_flag( TFLAG_LIQUID ) ) {
723 // water and lava
724 return false;
725 }
726 return true;
727}

764void vehicle::autodrive_controller::compute_goal_zone()
765{
766 data.goal_zone.clear();
767 coord_transformation goal_transform;
768 if( data.next_next_omt != data.next_omt ) {
769 // set the goal at the edge of next_omt and next_next_omt (in next_omt
770 // space, pointing towards next_next_omt)
771 const point next_omt_middle( OMT_SIZE + OMT_SIZE / 2, OMT_SIZE / 2 );
772 const tripoint_rel_omt omt_diff = data.next_next_omt - data.next_omt;
773 const quad_rotation rotation = to_quad_rotation( omt_diff.raw().xy() ) - data.nav_to_map.rotation;
774 goal_transform = {next_omt_middle, rotation, next_omt_middle};
775 } else {
776 // set the goal at the edge of cur_omt and next_omt (in next_omt space,
777 // pointing away from cur_omt)
778 goal_transform = { point( OMT_SIZE - 1, 0 ), quad_rotation::d0, point_zero };
779 }
780 constexpr int max_turns = NUM_ORIENTATIONS / 8 + 1;
781 const int x = 2 * OMT_SIZE - 1;
782 for( int turns = -max_turns; turns <= max_turns; turns++ ) {
783 const orientation dir = orientation::d0 + turns;
784 static_assert( NAV_MAP_SIZE_Y == OMT_SIZE, "Unexpected nav map size" );
785 for( int y = 0; y < OMT_SIZE; y++ ) {
786 const point pt( x, y );
787 const node_address addr = goal_transform.transform( pt, dir );
788 if( data.valid_position( addr ) ) {
789 data.goal_zone.insert( addr );
790 }
791 }
792 }
793 data.goal_points[0] = point( OMT_SIZE, OMT_SIZE / 2 - 1 );
794 data.goal_points[1] = goal_transform.transform( point( x, OMT_SIZE / 2 - 1 ) );
795}

847scored_address vehicle::autodrive_controller::compute_node_score( const node_address &addr,
848 const navigation_node &node ) const
849{
850 // TODO: tweak this
851 constexpr int cost_mult = 1;
852 constexpr int forward_dist_mult = 10;
853 constexpr int side_dist_mult = 8;
854 constexpr int angle_mult = 2;
855 constexpr int nearness_penalty = 15;
856 scored_address ret{ addr, 0 };
857 ret.score += cost_mult * node.cost;
858 if( node.is_goal ) {
859 return ret;
860 }
861 static const point neighbor_deltas[4] = { point_east, point_south, point_west, point_north };
862 for( const point &neighbor_delta : neighbor_deltas ) {
863 const point p = addr.get_point() + neighbor_delta;
864 if( !data.nav_bounds.contains( p ) || !data.valid_position( addr.facing_dir, p ) ) {
865 ret.score += nearness_penalty;
866 }
867 }
868 const int omt_index = std::min( std::max( addr.x / OMT_SIZE, 0 ),
869 static_cast<int>( data.goal_points.size() ) - 1 );
870 const point waypoint_delta = data.goal_points[omt_index] - addr.get_point();
871 const point goal_delta = data.goal_points.back() - addr.get_point();
872 const orientation waypoint_dir = approx_orientation( waypoint_delta.x, waypoint_delta.y );
873 ret.score += forward_dist_mult * std::abs( goal_delta.x );
874 ret.score += side_dist_mult * std::abs( goal_delta.y );
875 if( waypoint_delta.x > OMT_SIZE / 4 ) {
876 ret.score += angle_mult * std::abs( orientation_diff( addr.facing_dir, waypoint_dir ) );
877 }
878 return ret;
879}

882void vehicle::autodrive_controller::compute_next_nodes( const node_address &addr,
883 const navigation_node &node, int target_speed_tps,
884 std::vector<std::pair<node_address, navigation_node>> &next_nodes )
885const
886{
887 constexpr int move_cost = 10;
888 constexpr int steering_cost = 1;
889 const int sign = target_speed_tps > 0 ? 1 : -1;
890 const int target_speed = target_speed_tps * VMIPH_PER_TPS;
891 const int cur_omt = addr.x / OMT_SIZE;
892 int next_speed = target_speed;
893 int num_tiles_to_move = std::abs( target_speed_tps );
894 if( target_speed_tps > 1 && node.speed < target_speed ) {
895 const int cur_tps = std::min( std::max( node.speed / VMIPH_PER_TPS, 0 ), data.max_speed_tps - 1 );
896 next_speed = std::min( std::max<int>( node.speed, 0 ) + data.acceleration[cur_tps], target_speed );
897 num_tiles_to_move = next_speed / VMIPH_PER_TPS;
898 }
899 for( int steer = -data.max_steer; steer <= data.max_steer; steer++ ) {
900 node_address next_addr = addr;
901 next_addr.facing_dir = addr.facing_dir + steer;
902 tileray tdir = data.profile( next_addr.facing_dir ).tdir;
903 if( steer == 0 ) {
904 tdir.advance( node.tileray_steps );
905 }
906 bool ok = true;
907 bool goal_found = false;
908 for( int i = 0; i < num_tiles_to_move && ok; i++ ) {
909 tdir.advance( sign );
910 next_addr.x += tdir.dx();
911 next_addr.y += tdir.dy();
912 if( !data.nav_bounds.contains( next_addr.get_point() ) ) {
913 // it's ok to go out of bounds only from the "next" omt and only
914 // after crossing the goal zone
915 if( cur_omt == 0 || !goal_found ) {
916 ok = false;
917 }
918 } else if( !data.valid_position( next_addr ) ) {
919 ok = false;
920 } else if( !goal_found && data.goal_zone.find( next_addr ) != data.goal_zone.end() ) {
921 goal_found = true;
922 }
923 }
924 if( !ok ) {
925 continue;
926 }
927 const int next_omt = next_addr.x / OMT_SIZE;
928 if( next_omt < cur_omt ) {
929 // no going back to a previous omt
930 continue;
931 }
932 navigation_node next_node;
933 next_node.prev = addr;
934 next_node.tileray_steps = num_tiles_to_move + ( !steer ? node.tileray_steps : 0 );
935 // TODO: improve this
936 next_node.cost = node.cost + move_cost * num_tiles_to_move + ( steer ? steering_cost : 0 );
937 next_node.speed = next_speed;
938 next_node.target_speed_tps = target_speed_tps;
939 next_node.is_goal = goal_found;
940 next_nodes.emplace_back( next_addr, next_node );
941 }
942}

944cata::optional<std::vector<navigation_step>> vehicle::autodrive_controller::compute_path(
945 int speed_tps ) const
946{
947 if( speed_tps == 0 || speed_tps < -1 ) {
948 return cata::nullopt;
949 }
950 // TODO: tweak this
951 constexpr size_t max_search_count = 10000;
952 std::vector<navigation_step> ret;
953 // TODO: check simple reachability first and bail out or set upper bound on node score
954 std::unordered_map<node_address, navigation_node, node_address_hasher> known_nodes;
955 std::priority_queue<scored_address, std::vector<scored_address>, std::greater<>>
956 open_set;
957 const tripoint_abs_ms veh_pos = driven_veh.global_square_location();
958 const node_address start = data.nav_to_map.inverse().transform(
959 veh_pos.raw().xy(), to_orientation( driven_veh.face.dir() ) );
960 known_nodes.emplace( start, make_start_node( start, driven_veh ) );
961 open_set.push( scored_address{ start, 0 } );
962 int search_count = 0;
963 std::vector<std::pair<node_address, navigation_node>> next_nodes;
964 while( !open_set.empty() ) {
965 const node_address cur_addr = open_set.top().addr;
966 open_set.pop();
967 search_count++;
968 const navigation_node &cur_node = known_nodes[cur_addr];
969 if( cur_node.is_goal ) {
970 node_address addr = cur_addr;
971 while( !( addr == start ) ) {
972 const navigation_node &node = known_nodes.at( addr );
973 const node_address &prev = node.prev;
974 const tripoint_abs_ms prev_loc( data.nav_to_map.transform( prev.get_point(),
975 data.current_omt.z() ) );
976 ret.emplace_back( navigation_step{
977 prev_loc,
978 data.nav_to_map.transform( addr.facing_dir ),
979 node.target_speed_tps
980 } );
981 addr = prev;
982 }
983 return ret;
984 }
985 next_nodes.clear();
986 compute_next_nodes( cur_addr, cur_node, speed_tps, next_nodes );
987 for( const auto &next : next_nodes ) {
988 const node_address &next_addr = next.first;
989 const navigation_node &next_node = next.second;
990 auto iter = known_nodes.find( next_addr );
991 if( iter != known_nodes.end() ) {
992 const navigation_node &other_node = iter->second;
993 if( next_node.cost < other_node.cost ) {
994 const bool same_dir = cur_addr.facing_dir == next_addr.facing_dir;
995 const bool dir_multiple_45 = static_cast<int>( next_addr.facing_dir ) % 3 == 0;
996 if( ( !same_dir && next_node.tileray_steps == 1 ) || dir_multiple_45 ) {
997 iter->second = next_node;
998 }
999 }
1000 } else if( known_nodes.size() < max_search_count ) {
1001 known_nodes[next_addr] = next_node;
1002 open_set.push( compute_node_score( next_addr, next_node ) );
1003 }
1004 }
1005 }
1006 return cata::nullopt;
1007}

1026
1027collision_check_result vehicle::autodrive_controller::check_collision_zone( orientation turn_dir )
1028{
1029 const tripoint veh_pos = driven_veh.global_pos3();
1030
1031 // first check if we have any visibility in front, to prevent blind driving
1032 tileray face_dir = driven_veh.face;
1033 face_dir.advance();
1034 const point forward_offset( face_dir.dx(), face_dir.dy() );
1035 bool blind = true;
1036 for( const point &p : data.profile( to_orientation( face_dir.dir() ) ).collision_points ) {
1037 if( driver.sees( veh_pos + forward_offset + p ) ) {
1038 blind = false;
1039 }
1040 }
1041 if( blind ) {
1042 return collision_check_result::no_visibility;
1043 }
1044
1045 // now check the area we're about to move into in 1 step, anything there
1046 // should force us to stop
1047 const vehicle_profile &profile = data.profile( turn_dir );
1048 tileray tdir = driven_veh.face;
1049 if( turn_dir != to_orientation( tdir.dir() ) ) {
1050 tdir = profile.tdir;
1051 }
1052 const int speed = std::min( driven_veh.velocity + driven_veh.current_acceleration(),
1053 driven_veh.cruise_velocity );
1054 const int speed_tps = speed / VMIPH_PER_TPS;
1055 std::unordered_set<point> collision_zone;
1056 tdir.advance();
1057 point offset( tdir.dx(), tdir.dy() );
1058 for( const point &p : profile.occupied_zone ) {
1059 collision_zone.insert( p + offset );
1060 }
1061 for( const point &p : collision_zone ) {
1062 if( !check_drivable( veh_pos + p ) ) {
1063 return collision_check_result::close_obstacle;
1064 }
1065 }
1066
1067 // finally check the area further ahead; we can still avoid those collisions by reducing speed
1068 collision_zone.clear();
1069 for( int i = 1; i < speed_tps; i++ ) {
1070 tdir.advance();
1071 offset += point( tdir.dx(), tdir.dy() );
1072 for( const point &p : profile.collision_points ) {
1073 collision_zone.insert( p + offset );
1074 }
1075 }
1076 for( const point &p : collision_zone ) {
1077 if( !driver.sees( veh_pos + p ) ) {
1078 return collision_check_result::slow_down;
1079 }
1080 if( !check_drivable( veh_pos + p ) ) {
1081 return collision_check_result::slow_down;
1082 }
1083 }
1084 return collision_check_result::ok;
1085}

1091
1092cata::optional<navigation_step> vehicle::autodrive_controller::compute_next_step()
1093{
1094 precompute_data();
1095 const tripoint_abs_ms veh_pos = driven_veh.global_square_location();
1096 const bool had_cached_path = !data.path.empty();
1097 while( !data.path.empty() && data.path.back().pos != veh_pos ) {
1098 data.path.pop_back();
1099 }
1100 if( !data.path.empty() && data.path.back().target_speed_tps > data.max_speed_tps ) {
1101 data.path.clear();
1102 }
1103 if( data.path.empty() ) {
1104 // if we're just starting out or we've gone off-course use the lowest speed
1105 if( had_cached_path || driven_veh.velocity == 0 ) {
1106 data.max_speed_tps = MIN_SPEED_TPS;
1107 }
1108 auto new_path = compute_path( data.max_speed_tps );
1109 while( !new_path && data.max_speed_tps > MIN_SPEED_TPS ) {
1110 // high speed didn't work, try a lower speed
1111 data.max_speed_tps /= 2;
1112 new_path = compute_path( data.max_speed_tps );
1113 }
1114 if( !new_path ) {
1115 return cata::nullopt;
1116 }
1117 data.path.swap( *new_path );
1118 }
1119 return data.path.back();

1210 }

1212 driver.add_msg_if_player( m_warning, _( "You lose control as the vehicle starts skidding." ) );
1213 stop_autodriving( false );

1215 }
1216 const tripoint_abs_ms veh_pos = global_square_location();
1217 const tripoint_abs_omt veh_omt = project_to<coords::omt>( veh_pos );
1218 std::vector<tripoint_abs_omt> &omt_path = driver.omt_path;
1219 if( !omt_path.empty() && veh_omt == omt_path.back() ) {
1220 omt_path.pop_back();
1221 }
1222 if( omt_path.empty() ) {
1223 stop_autodriving( false );
1224 return autodrive_result::finished;
1225 }
1226 if( !active_autodrive_controller ) {
1227 active_autodrive_controller = std::make_shared<autodrive_controller>( *this, driver );
1228 }
1229 if( &active_autodrive_controller->get_driver() != &driver ) {
1230 debugmsg( "Driver changed while auto-driving" );
1231 stop_autodriving();

1233 }
1234 active_autodrive_controller->check_safe_speed();
1235 cata::optional<navigation_step> next_step = active_autodrive_controller->compute_next_step();
1236 if( !next_step ) {
1237 // message handles pathfinding failure either due to obstacles or inability to see
1238 driver.add_msg_if_player( _( "Can't see a path forward." ) );
1239 stop_autodriving( false );
1240 return autodrive_result::abort;
1241 }
1242 if( next_step->pos != veh_pos ) {
1243 debugmsg( "compute_next_step returned an invalid result" );
1244 stop_autodriving();
1245 return autodrive_result::abort;
1246 }
1247 if( next_step->target_speed_tps == 0 && velocity == 0 ) {
1248 stop_autodriving( false );
1249 return autodrive_result::finished;
1250 }
1251 cruise_on = true;
1252 cruise_velocity = next_step->target_speed_tps * VMIPH_PER_TPS;
1253
1254 // check for collisions before we steer, since steering may end our turn
1255 // which would cause the vehicle to move and maybe crash
1256 switch( active_autodrive_controller->check_collision_zone( next_step->steering_dir ) ) {
1257 case collision_check_result::no_visibility:
1258 driver.add_msg_if_player( m_warning, _( "You can't see anything in front of your vehicle!" ) );
1259 stop_autodriving();

1261 case collision_check_result::close_obstacle:
1262 driver.add_msg_if_player( m_warning, _( "You're about to crash into something!" ) );
1263 stop_autodriving( false );
1264 return autodrive_result::abort;
1265 case collision_check_result::slow_down:
1266 active_autodrive_controller->reduce_speed();
1267 if( cruise_velocity > VMIPH_PER_TPS ) {
1268 cruise_velocity = VMIPH_PER_TPS;
1269 }
1270 break;
1271 case collision_check_result::ok:
1272 break;
1273 }
1274
1275 int turn_delta = orientation_diff( next_step->steering_dir, to_orientation( turn_dir ) );
1276 // pldrive() does not handle steering multiple times in one call correctly
1277 // call it multiple times, matching how a player controls the vehicle
1278 for( int i = 0; i < std::abs( turn_delta ); i++ ) {
1279 if( driver.moves <= 0 ) {
1280 // we couldn't steer as many times as we wanted to but there's
1281 // nothing we can do about it now, hope we don't crash!
1282 break;
1283 }
1284 pldrive( driver, { signum( turn_delta ), 0 } );
1285 }
1286 // Don't do anything else below; the driver's turn may be over (moves <= 0) so
1287 // any extra actions would be "cheating".
1288
1289 return autodrive_result::ok;
1290}
