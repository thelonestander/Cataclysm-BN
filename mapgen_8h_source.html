130/**
131 * Basic mapgen object. It is supposed to place or do something on a specific square on the map.
132 * Inherit from this class and implement the @ref apply function.
133 *
134 * Instructions for adding more mapgen things:
135 * 1. create a new class inheriting from @ref jmapgen_piece.
136 * - It should have a constructor that a accepts a @ref JsonObject and initializes all it local
137 * members from it. On errors / invalid data, call @ref JsonObject::throw_error or similar.
138 * - Implement the @ref apply function to do the actual change on the map.
139 * 2. Go into @ref mapgen_function_json::setup and look for the lines with load_objects, add a new
140 * line with your own class there. It should look like
141 * @code load_objects<your_own_class_from_step_1>( jo, "place_something" ); @endcode
142 * Use a descriptive name for "something", preferably matching the name of your class.
143 * 3. Go into @ref mapgen_function_json::setup and look for the lines with load_place_mapings, add
144 * a new line with your class there. It should look like
145 * @code load_place_mapings<your_own_class_from_step_1>( jo, "something", format_placings ); @endcode
146 * Using the same "something" as in step 2 is preferred.
147 *
148 * For actual examples look at the commits that introduced the load_objects/load_place_mapings
149 * lines (ignore the changes to the json files).
150 */

152{
153 protected:
154 jmapgen_piece() : repeat( 1, 1 ) { }
