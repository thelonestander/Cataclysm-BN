26// The behavior tree is (at least initially) intended to decide on a goal for a given subject.
27// Each time tick is called, it evaluates the tree to find the highest-priority behavior that is
28// capable of making forward progress.
29// The behavior tree performs a depth-first traversal until it reaches an unmet goal.
30// When tick is invoked, it visits root -> node_t::tick -> strategy -> children -> node_t::tick
31// At each level, the strategy determines the order in which to visit the current node's children.
32// Once a leaf node is reached, it either returns success, indicating it's requirements are met,
33// failure, indicating that it is incapable of satisfying it's requirements, or running, indicating
34// that it is capable of addressing tis requirements, but that they aren't met yet.
35// In practice, the tree is traversed until it reaches a node that returns running,
36// meaning that it is capable of making progress if set as a goal.
37// The arrangement of the tree and configuration of iteration strategies guarantees that it visits
38// nodes in order of descending priority.
39// A predicate can be injected into any node to direct iteration,
40// including skipping entire subtrees.
