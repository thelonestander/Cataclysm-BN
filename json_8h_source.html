96/* JsonIn
97 * ======
98 *
99 * The JsonIn class provides a wrapper around a std::istream,
100 * with methods for reading JSON data directly from the stream.
101 *
102 * JsonObject and JsonArray provide higher-level wrappers,
103 * and are a little easier to use in most cases,
104 * but have the small overhead of indexing the members or elements before use.
105 *
106 * Typical JsonIn usage might be something like the following:
107 *
108 * JsonIn jsin(myistream);
109 * // expecting an array of objects
110 * jsin.start_array(); // throws JsonError if array not found
111 * while (!jsin.end_array()) { // end_array returns false if not the end
112 * JsonObject jo = jsin.get_object();
113 * ... // load object using JsonObject methods
114 * }
115 *
116 * The array could have been loaded into a JsonArray for convenience.
117 * Not doing so saves one full pass of the data,
118 * as the element positions don't have to be read beforehand.
119 *
120 * If the JSON structure is not as expected,
121 * verbose error messages are provided, indicating the problem,
122 * and the exact line number and byte offset within the istream.
123 *
124 *
125 * Single-Pass Loading
126 * -------------------
127 *
128 * A JsonIn can also be used for single-pass loading,
129 * by passing off members as they arrive according to their names.
130 *
131 * Typical usage might be:
132 *
133 * JsonIn jsin(&myistream);
134 * // expecting an array of objects, to be mapped by id
135 * std::map<std::string,my_data_type> myobjects;
136 * jsin.start_array();
137 * while (!jsin.end_array()) {
138 * my_data_type myobject;
139 * jsin.start_object();
140 * while (!jsin.end_object()) {
141 * std::string name = jsin.get_member_name();
142 * if (name == "id") {
143 * myobject.id = jsin.get_string();
144 * } else if (name == "name") {
145 * myobject.name = _(jsin.get_string());
146 * } else if (name == "description") {
147 * myobject.description = _(jsin.get_string());
148 * } else if (name == "points") {
149 * myobject.points = jsin.get_int();
150 * } else if (name == "flags") {
151 * if (jsin.test_string()) { // load single string tag
152 * myobject.tags.insert(jsin.get_string());
153 * } else { // load tag array
154 * jsin.start_array();
155 * while (!jsin.end_array()) {
156 * myobject.tags.insert(jsin.get_string());
157 * }
158 * }
159 * } else {
160 * jsin.skip_value();
161 * }
162 * }
163 * myobjects[myobject.id] = myobject;
164 * }
165 *
166 * The get_* methods automatically verify and skip separators and whitespace.
167 *
168 * Using this method, unrecognized members are silently ignored,
169 * allowing for things like "comment" members,
170 * but the user must remember to provide an "else" clause,
171 * explicitly skipping them.
172 *
173 * If an if;else if;... is missing the "else", it /will/ cause bugs,
174 * so preindexing as a JsonObject is safer, as well as tidier.
175 */

552 *
553 * The JsonOut class provides a straightforward interface for outputting JSON.

555 * It wraps a std::ostream, providing methods for writing JSON data directly.

758 * ==========
759 *
760 * The JsonObject class provides easy access to incoming JSON object data.
761 *
762 * JsonObject maps member names to the byte offset of the paired value,
763 * given an underlying JsonIn stream.
764 *
765 * It provides data by seeking the stream to the relevant position,
766 * and calling the correct JsonIn method to read the value from the stream.
767 *
768 *
769 * General Usage
770 * -------------
771 *
772 * JsonObject jo(jsin);

958/* JsonArray
959 * =========
960 *
961 * The JsonArray class provides easy access to incoming JSON array data.
962 *
963 * JsonArray stores only the byte offset of each element in the JsonIn stream,
964 * and iterates or accesses according to these offsets.
965 *
966 * It provides data by seeking the stream to the relevant position,
967 * and calling the correct JsonIn method to read the value from the stream.
968 *
969 * Arrays can be iterated over,
970 * or accessed directly by element index.
971 *
972 * Elements can be returned as standard data types,
973 * or automatically read into a compatible container.
974 *
975 *
976 * Iterative Access
977 * ----------------
978 *
979 * JsonArray ja = jo.get_array("some_array_member");
980 * std::vector<int> myarray;
981 * while (ja.has_more()) {
982 * myarray.push_back(ja.next_int());
983 * }
984 *
985 * If the array member is not found, get_array() returns an empty array,
986 * and has_more() is always false, so myarray will remain empty.
987 *
988 * next_int() and the other next_* methods advance the array iterator,
989 * so after the final element has been read,
990 * has_more() will return false and the loop will terminate.
991 *
992 * If the next element is not an integer,
993 * JsonArray will throw a JsonError indicating the problem,
994 * and the position in the input stream.
995 *
996 * To handle arrays with elements of indeterminate type,
997 * the test_* methods can be used, calling next_* according to the result.
998 *
999 * Note that this style of iterative access requires an element to be read,
1000 * or else the index will not be incremented, resulting in an infinite loop.
1001 * Unwanted elements can be skipped with JsonArray::skip_value().
1002 *
1003 *
1004 * Positional Access
1005 * -----------------
1006 *
1007 * JsonArray ja = jo.get_array("xydata");
1008 * point xydata(ja.get_int(0), ja.get_int(1));
1009 *
1010 * Arrays also provide has_int(index) etc., for positional type testing,
1011 * and read(index, &container) for automatic deserialization.
1012 *
1013 *
1014 * Automatic Deserialization
1015 * -------------------------
1016 *
1017 * Elements can also be automatically read into compatible containers,
1018 * such as maps, sets, vectors, and classes implementing JsonDeserializer,
1019 * using the read_next() and read() methods.
1020 *
1021 * JsonArray ja = jo.get_array("custom_datatype_array");
1022 * while (ja.has_more()) {
1023 * MyDataType mydata; // MyDataType implementing JsonDeserializer
1024 * ja.read_next(mydata);

1026 * }
